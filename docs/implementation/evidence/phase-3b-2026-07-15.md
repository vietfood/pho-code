## Implemented boundary

- `pho chat` owns one alternate-screen terminal session with a bounded in-process display transcript and accepts repeated independent ephemeral turns.
- `pho chat --raw` and `pho chat --stdin` retain one-shot raw projections without cursor-control output.
- The TUI derives its view model from canonical runtime events, while the reducer and coordinator remain authoritative for lifecycle and terminal truth.
- Provider-returned reasoning remains collapsed by default and is shown only through explicit disclosure.
- Input, transcript, redraw, renderer, and terminal operations are bounded; presentation failure becomes cancellation input.
- The terminal guard restores raw mode, descriptor flags, alternate-screen state, bracketed paste, and cursor visibility on exit paths covered by the deterministic and PTY checks.
- Displayed prior turns do not enter later model requests. Phase 3B adds no durable conversation, journal, recovery, or ordinary-workspace tool behavior.
