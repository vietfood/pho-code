## Implemented boundary

- The canonical domain has distinct local workspace, session, turn, item, backend-request, tool-call, approval, and artifact identities plus exhaustive turn, tool, approval, and terminal state classifications. Provider completion/call identities remain separate.
- The context builder preserves complete assistant text/reasoning/call phases and exact ordered tool results, rejects missing or duplicate identities, invalid/incomplete arguments, profile switches, and size overflow, and reports `TooLarge` without truncation.
- The general turn loop retains the Phase 1B no-tool/live helpers while adding fixed scripted schemas, sequential calls, strict prevalidation, at-most-once execution, effect-bound approval responses, explicit denial/unavailable results, exact continuation replay, and one terminal outcome.
- Configured context, continuation, tool-count, argument, result, approval, duration, backend-event, canonical-event, and presentation-event limits stop before the next request/effect and project a visible limit result. Nonzero backend channels remain bounded; closed/failing presentation cancels the owner.
- `ScriptedBackend` records normalized requests and supports deterministic events, failure after prior events, and cancellation acknowledgement without exposing a release command selector.
- The reducer retains ordered assistant phases, tool and approval lifecycle state, invalidates pending work on terminal outcomes, rejects stale approvals, and keeps terminal turns immutable.
- The terminal renderer projects reasoning, text, tools, approvals, results, usage, limits, failure, and cancellation. Runtime/agent debug output redacts content-bearing events.

No real filesystem/process tool, session journal, crash recovery, GPUI view, provider DTO, custom endpoint, or release-surface backend selector was added.
