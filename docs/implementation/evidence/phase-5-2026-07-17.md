## Implemented boundary

Sessions use a private, versioned JSONL journal under Pho Code's Application Support directory. One writer owns sequence allocation and flushes every user-message, backend-request, approval, execution, effect-progress, paired result, usage, and terminal-turn boundary required for restart safety. Completed assistant phases retain text, bounded provider-returned reasoning, replay requirement, ordered calls, and provider call identities. The runtime reconstructs complete canonical history and sends it through the existing backend and reducer; it neither truncates old history nor resumes an interrupted request automatically.

Persistent artifacts use opaque identities, per-session ownership, bounded accounting, a same-directory temporary file, file flush, atomic rename, and directory flush. Production limits are 2 MiB per artifact, 64 MiB per session, and 1 GiB globally. Journals are bounded to 1,000,000 records and 256 MiB, with 512 KiB payloads and 1 MiB lines. Directories and files are reset to user-only permissions on the supported host.

Recovery distinguishes clean journals, recoverable non-newline tails, malformed complete lines, sequence corruption, and unsupported required schemas. Torn-tail repair retains the intact damaged original and activates a flushed valid-prefix replacement with a recovery diagnostic. Open requests and unresolved approvals become interrupted without replay. A started effect without matching completion records the exact uncertain path; completed execution without a paired model-visible result remains interrupted and is neither synthesized nor rerun.

Ordinary `pho chat` now creates a durable session for the selected personal workspace. `pho session list` operates offline, `pho session resume <session-id>` accepts only a bounded opaque UUID, and reconstructed, interrupted, uncertain, read-only, missing-workspace, and provider-reasoning state use the canonical terminal projection. The explicitly named loopback/developer fixtures remain available without changing the release command boundary.
