## Implemented boundary

- Ordinary `pho chat` now constructs the shared coordinator with the Phase 2 general loop instead of the no-tool helper. The loop receives two fixed process-local Phase 3 schemas: the already qualified in-memory echo tool and a fake mutating probe with no executable local effect. The command policy always denies the fake mutation; real workspace effects remain absent until Phase 4.
- Full-history request projection retains grouped assistant text, provider-returned reasoning, ordered calls, exact provider call IDs, and paired tool results. The concrete DeepSeek adapter validates qualified model/profile identity, local request identity at stream start and completion, request/message/tool/schema bounds, strict replay arguments, and stable completion/call identities before send or continuation.
- Approval decisions are asynchronous and cancellation-aware. Cancellation while waiting invalidates the pending operation and reaches one terminal outcome without executing the fake mutation.
- The loop accepts cancellation only when the backend reports local transport termination. Reasoning, text, tool-argument, approval-wait, and continuation-request cancellation paths are covered through the real HTTP/SSE adapter; process SIGINT and broken stdout are covered while that adapter owns an active stream.
- Length, content-filter, resource-interruption, invalid request, rejected request, invalid credential, insufficient balance, unavailable model, rate limit, service failure, malformed/oversized SSE, early stream end, incompatible choice/event, missing finish, replay failure, and unacknowledged cancellation retain redacted typed outcomes. Semantic chunks after a finish reason fail closed; terminal cache and reasoning usage must agree with prompt and completion usage. HTTP failure fixtures assert one request and no automatic retry.
- Prompt, cache-hit, cache-miss, output, reasoning, and total usage accumulate across model continuations. Estimated cost is labeled as an estimate, identifies the observation date and authoritative provider ledger, and becomes unknown when usage is incomplete, the price profile is over 30 UTC days old, or the local clock predates it.
- A debug-only loopback constructor accepts only numeric `http` loopback addresses at `/chat/completions` and owns a fixed synthetic fixture key; its type cannot receive, lease, or invalidate the production credential actor. The command adapter also honors its test endpoint only with the in-memory test credential store. Production construction remains fixed to the allowlisted DeepSeek origin with redirects disabled.
- Command stdout and stderr each have a 16 MiB renderer budget and use nonblocking descriptors whose prior flags are restored on drop. The current command projection drains canonical and presentation events synchronously, so neither layer can build an asynchronous backlog; a zero configured handoff capacity fails before starting work. An exhausted renderer budget, non-draining pipe, detached terminal, or broken pipe cancels the active stream and becomes one visible terminal outcome instead of blocking the coordinator.

No filesystem/process tool, arbitrary production endpoint, provider discovery, automatic retry, session persistence, beta API feature, or GPUI path was added.
