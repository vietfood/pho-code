## Implemented boundary

- The shared loop advertises `search_files`, `search_text`, `read_file`, `apply_patch`, and `shell` only through the Phase 4 runtime. Actual effects are debug/developer-only and require a canonical subdirectory of the operating-system temporary directory; ordinary personal workspaces remain unavailable until Phase 5 supplies durable journals, artifacts, and recovery.
- Complete strict arguments are prepared before execution. Each prepared operation declares a conservative model-result ceiling, and loop limits, cancellation, duplicate call identity, exact approval identity, denial, terminal status, and uncertain effects remain canonical application events.
- Workspace access retains the canonical root directory descriptor and device/inode identity. Structured traversal uses descriptor-relative `openat` with `O_NOFOLLOW`; pathname/descriptor root divergence, lexical escapes, symlinks, ancestor retargeting, stale source identity, and shell-cwd replacement fail closed.
- Search uses the pinned `fff-search` 0.9.6 in-memory AI picker and fuzzy ranking. The built-in watcher, mmap/content cache, and grep reader remain disabled. An owned `notify` dirty-signal adapter revalidates root identity and the 100,000-file ignore-aware preflight before scheduling no-follow FFF rescans; content matching reopens candidates through the workspace descriptor and reports skip counters.
- Reads return stable numbered UTF-8 windows under independent file, line, and result limits. Device/inode/length/mtime/ctime identity is checked on the read descriptor and a descriptor-safe pathname reopen.
- Patch uses an independently implemented Begin Patch subset. Codex-style `@@ <locator>` headers are accepted for model compatibility, but every hunk body is matched exactly and uniquely against the original source; overlapping or dependent hunks fail. Writes use descriptor-relative unguessable siblings, fsync, metadata validation, macOS atomic rename/swap, recoverable Trash quarantine, complete recovery envelopes, directional effect progress, cancellation rollback, and exact uncertain paths.
- Shell runs `/bin/zsh -f -c` with closed stdin, a cleared small environment, a descriptor-bound cwd, a new process group, timeout/cancellation cleanup, bounded byte capture, lossy-preview metadata, and opaque artifact references. Every command requires a fresh exact approval. The conservative permanent-deletion classifier covers reviewed direct, absolute, assignment, redirection, wrapper, `find -exec`, and `find -delete` forms; approval rendering escapes terminal controls and still states that the shell is not a sandbox.
